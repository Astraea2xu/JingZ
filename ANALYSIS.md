# 小云雀安装包分析与独立实现说明

## 结论

所分析的 `xiaoyunque` 是 Electron 28 桌面客户端，安装包版本为 1.0.51。它不是把完整 AI Agent 放在本机运行，而是把在线站点包装为桌面应用：

- 主窗口加载 `https://xyq.jianying.com`，并添加 `utm_medium=internal_diversion` 与 `utm_source=xyq_native`。
- 本地进程负责窗口生命周期、登录回调、外部链接隔离、应用菜单、快捷键和自动更新。
- 渲染进程启用了 `contextIsolation`、关闭 `nodeIntegration`、启用沙箱和 Web 安全。
- 本地 preload 只暴露受限 IPC：应用信息、更新流程、创作菜单同步、菜单命令。
- 本地创作菜单包含新建创作、短剧 Agent、创作历史；具体工具与模板可由网页动态下发。
- 更新由 `@ies/tron-client` 与 `electron-updater` 共同处理，并生成持久化的匿名设备 ID。

因此，安装包本身无法提供在线端的模型提示词、工作流实现或生成 API。直接复制这个 Electron 壳，只会得到一个依赖原站点的浏览器窗口。

## 可确认的产品能力

结合安装包菜单和公开产品页，可以归纳出这些核心创作面：

1. 一句话/素材驱动的智能成片。
2. 短剧 Agent：从故事或剧本到角色、分镜和成片。
3. 数字人口播。
4. AI 图片与视觉设计。
5. 对参考内容做结构分析，再生成新的内容。
6. 创作历史、多端项目和桌面菜单快捷入口。

## 镜舟的实现

镜舟复刻的是“创作编排能力”，不复制小云雀品牌、代码、私有接口或模型：

| 能力 | 镜舟首版 |
|---|---|
| 模糊需求理解 | 已实现 |
| 剧本与情绪节拍 | 已实现 |
| 角色视觉锚点 | 已实现 |
| 逐镜头分镜 | 已实现 |
| 图像/视频提示词 | 已实现 |
| 发布文案与封面方案 | 已实现 |
| 本地项目历史 | 已实现 |
| 文本模型接入 | OpenAI-compatible |
| 图像模型接入 | 已预留接口 |
| Seedance 2.0 视频任务 | 已实现创建、轮询和错误记录 |
| 音画同生 | 已实现 Seedance `generate_audio` 配置 |
| 镜头连续性 | 已实现上一段尾帧接力 |
| 视频永久保存 | 已实现成功后自动下载到本地 |
| 自动合成成片 | 已实现，运行环境需提供 FFmpeg |
| 数字人私域素材 | 未实现，需开通对应资产权限 |

## 下一步推荐

当前主链已经支持 Seedance 2.0。下一步可继续补充：

1. 用 Seedream 自动生成角色定妆图，再作为 Seedance 参考输入。
2. 对失败镜头提供局部改写和单独重试。
3. 接入私域真人/数字人资产库。
4. 在合片阶段加入独立 TTS、字幕时间轴与全片统一配乐。
